Let's talk about sharing links and possible improvements:


# Share links / refresh needed: 

There is an issue when I open a link via QR code on iPhone. It correctly opens the browser, and switches to the tab where it was opened. However I do have to click reload for the score to appear. Otherwise I remain on the same screen where I was.

# Using links to share games database: 
When sharing a game link, it would be nice if it was saved inside the receiving user database.
- Games would need to have a uuid to enable updates.
- A game received like this would be marked "View Only".
- When opening a "View Only" game, the user would have the option to "Become the score keeper" and convert it to a normal game in their database. This way they could take turns keeping score, or edit the game if they notice a mistake.
- When receiving a game link for a game for which the user is already the score keeper, they would be informed that a new version of the game is available, and that they may update the scores (and they will not be the score keeper anymore).
- When sharing a link, we could have a differently worded version such as "Change score keeper" which would make the current user a "View Only" user and the receiving user the new score keeper.

# Share link to "installed app": 
When opening a share link, if the app is installed, it would be nice if it opened the app instead of the browser. This way the user would have a better experience and could easily save the game to their database. Not sure if this is possible with a PWA.

# Copy link:

At the moment the copy link button opens a dialogue where you can manually copy the text. It would be better if it placed it inside the clipboard automatically.
